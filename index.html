HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Declarations are not case-sensitive; HTML elements are case sensative.
  • HTML attributes are always defined in the beginning element.
  • Metadata: data about our webpage that the browser uses to effectively serve the webpage.
  • Style element: defines the CSS styles associated with a website
  • Base element: defines the base URL for a webpage
  • Link element: connects an external resource to the HTML document.
  • Meta element: defines metadata such as character set, descriptions, keywords, author, and viewport
  • Script element: embeds client-side scripts in the HTML document (same as link element but for JavaScript).
  • Element: refers to the entire element from opening to closing tag.
  • Tag: refers to the what appears inside the angle brackets
  • Headings have levels h1-h6.
  • ul element: unordered list
  • ol element: ordered list
  • src attribute : source/location of image
  • a element: anchor element - creates links to the same/other webpages
  • href attribute: points to the URL for the link.

CSS

    • A margin indicates how much space we want around the outside of an element.
    • A padding indicates how much space we want around the content inside an element.
    • There are 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
    • Internal CSS rules will be in the head section of the HTML file.
    • When using an external style sheet a link element must be placed in the HTML's head.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds the changed files in our working directory to the staging area
  • git commit -m "Example Message Here": commits changes to our repository and adds concise message to commit of changes
  • git pull origin main: makes sure our current branch is up to date
  • git push origin feature/Example-Branch-Name: pushes our changes to the main branch
  • git checkout main: switches to the main git branch

JavaScript

  • Use script element to apply JavaScript to Page. Added within the body element
  • Semicolons indicate the end of a statement.
  • JavaScript is case sensative.
  • var: use this keyword to declare a variable.
  • console.log: will out a message to the web console.
  • Indexes are the unique location number for each piece of data in an array, they start with 0.
  • Array: a single variable that is used to hold a group of data, that is typically related.
  • for loop: tells the computer to go through each piece of data in an array, running the code again until complete.
  • function: a set of instructions that tells the computer how to perfom a certain task.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
